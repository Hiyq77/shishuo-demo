# 02 用户角色与权限

## 4. 用户与角色

### 4.1 角色（MVP）


| 角色              | 核心诉求                   |
| --------------- | ---------------------- |
| **任课教师**        | 写教案、参与教研、被评课或自评、管理个人资源 |
| **备课组长 / 教研组长** | 发起教研、汇总成果、组织听评课（若权限允许） |
| **校管理者（可选占位）**  | 查看有限列表与汇总（MVP 可极弱）     |


### 4.2 用户资料与展示（首页）

- 首页顶栏需展示：**姓名、角色、所属学校**；其中「所属学校」与账号/租户绑定，单校部署时可为常量或组织名。
- **本期数据来源约定**：上述用户资料字段（含姓名、角色、所属学校及扩展档案）均通过调用需求方提供的 API 获取；**C 端业务不提供用户资料维护能力**（不设编辑页/管理页）。

### 4.3 权限（MVP 简化）

- 默认 **同校可见**（若仅单校部署，可先不做跨校）。  
- 资源可见性：**仅本人 / 校内公开（占位）** 二选一或简化为「创建人可编辑，同角色只读」——需在实现前定一条默认策略（PRD 建议：**默认本人，可显式分享到教研主题参与者**）。
