# 05 全链路闭环

## 7. 「全链路」闭环（MVP 可落地的关联）


| 关联      | MVP 要求                   |
| ------- | ------------------------ |
| 教案 ↔ 教研主题 | 均在 **协作教研** 应用内；教案可选「关联教研主题」（下拉选择进行中的主题） |
| 评课 ↔ 教案 | 评课表单可选「关联教案」（若存在）        |
| 统一沉淀    | 所有产出进资源库，带类型与来源 ID       |


**说明**：MVP 不强制做自动「改进建议回流教案」，但 UI 可提供「从评课打开关联教案」以完成人工闭环。

## 核心流程（User Flows -> 核心流程）

### 流程 A：协作教研（主题、纪要、教案、课堂笔记）

- 触发：教师或组长进入 **协作教研** 应用。  
- 核心步骤（可并行、非必须顺序）：  
  - **教研主题**：创建主题 → 讨论协作 → AI 生成纪要 → 纪要入库；  
  - **教案**：新建向导 → 生成初稿 → 富文本改写 → 草稿/定稿 → 教案入库；  
  - **课堂笔记**：新建笔记 → 可选关联主题或教案 → 保存。  
- 输入：主题信息、讨论内容、教案字段、笔记正文、资料链接。  
- 输出：结构化纪要、教案正文、笔记对象、资源库条目（含来源对象 ID）。

### 流程 B：课堂观察与改进回流

- 触发：教师/组长创建听评课任务或自评任务。  
- 核心步骤：填写课例与证据 → AI 生成报告 → 报告入库 → 从报告跳转关联教案（位于 **协作教研** 应用）以完成人工改进。  
- 输入：量表分值、课堂记录、视频评价结果。  
- 输出：结构化评课报告、改进建议、资源库条目（含来源评课 ID）。

## 跨流程状态流转（MVP）

- 业务对象状态：草稿 -> 已提交/已定稿 -> 已入库。  
- AI 任务状态：待执行 -> 生成中 -> 成功/失败。  
- 关联状态：未关联 -> 已关联（教研-教案、教案-评课）。

## 跨流程边界情况（MVP）

- 关联对象不存在或被归档时，保留历史引用并提示不可编辑。  
- 任一环节 AI 失败时不阻断人工流程，允许人工填写并继续入库。  
- 入库失败时保留业务对象原文与生成结果，支持延迟重试。  
- 用户权限不足时允许查看关联信息，但禁止修改源对象。

## 全链路数据埋点（MVP）

- `flow_research_to_repo_completed`：完成教研主题到纪要入库闭环。  
- `flow_lesson_plan_to_repo_completed`：完成教案到入库闭环。  
- `flow_evaluation_to_repo_completed`：完成评课到报告入库闭环。  
- `flow_cross_link_created`：创建跨对象关联（教研-教案或教案-评课）。  
- `flow_cross_link_opened`：从一个对象跳转打开关联对象。
