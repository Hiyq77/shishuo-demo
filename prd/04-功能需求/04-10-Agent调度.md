# 04-10-Agent调度

### 6.10 智能识别与 Agent 调度

#### 目标

- 对用户自然语言输入进行任务意图识别，自动选择更合适的处理方式（对话问答、文稿编辑、结构化生成）。  
- 在可识别场景下自动调度专职 Agent，提升结果质量与任务完成效率。

#### 识别与路由规则（MVP）

- 系统需对输入进行基础任务分类：  
  - `对话模式`：讨论教学思路、追问细节、即时答疑  
  - `文稿编辑`：围绕当前文档补写、润色、改写、整理  
  - `结构化生成`：按固定模板输出（如教案、评课报告、练习题）  
- 当识别置信度高于阈值时，自动执行路由；低于阈值时，提示用户选择处理方式。  
- 路由结果需在 UI 中可感知（显示当前已调用的能力或 Agent）。

#### Agent 能力清单（MVP，仅 3 类）

- **教案 Agent**：协作教研应用内——教案初稿生成、分段改写、教学活动与作业补充；教研主题提纲、纪要、行动项等结构化生成。  
- **课堂观察 Agent**：课堂观察应用内——课堂记录总结、简评与改进建议、结构化观察报告、围绕视频时间锚点的解释与追问辅助。  
- **通用对话 Agent**：首页「师说」助手与各应用内兜底——低置信度路由确认、目标 Agent 不可用/失败后的回退、跨应用意图澄清与导航提示。

#### 后续版本 Agent（非 MVP）

- 导图 Agent、图谱 Agent、课件 Agent、练习 Agent 等：在教研资源生成、知识关系可视化、课件纲要、习题分层等场景扩展；本期仅保留名称与方向，不在实现范围与验收范围内。

#### 调度与回退机制

- 默认由统一调度层按任务类型分发到对应 Agent。  
- 若目标 Agent 不可用或执行失败，自动回退到通用模型并提示用户。  
- 用户可手动切换 Agent 或改用通用对话模式。  
- 所有调度请求需记录任务类型、路由结果、耗时与状态，用于后续优化。

#### 验收口径

- 至少 3 类任务类型（对话、文稿编辑、结构化生成）可自动识别并正确路由。  
- **上述 3 类 Agent** 均有至少 1 个可用场景，且输出可回填到当前业务对象（或首页助手可完成跨应用跳转说明）。  
- 置信度低场景下可触发用户确认，不出现静默错误路由。  
- Agent 失败后可回退到通用模型，流程不中断。


**输入/输出**：

- 输入：用户指令、上下文摘要、路由阈值、Agent 可用性。  
- 输出：任务分类结果、路由目标、Agent 产出内容、回退结果。

**状态流转**：

- 路由状态：待识别 -> 已识别 -> 已路由/待用户确认。  
- 执行状态：执行中 -> 成功/失败 -> 回退成功/回退失败。

**边界情况**：

- 低置信度时必须弹出模式选择，不自动静默路由。  
- Agent 不可用时自动回退通用模型并提示原因。  
- 回退也失败时保留请求上下文供再次执行。

**数据埋点（MVP）**：

- `agent_route_classified`：完成任务分类。  
- `agent_route_confirmed`：用户确认路由模式。  
- `agent_execution_started`：开始调用 Agent。  
- `agent_execution_succeeded`：Agent 执行成功。  
- `agent_execution_failed`：Agent 执行失败。  
- `agent_fallback_triggered`：触发通用模型回退。
