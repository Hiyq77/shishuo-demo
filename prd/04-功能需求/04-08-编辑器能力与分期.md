# 04-08-编辑器能力与分期

### 6.8 编辑器能力范围与分期（Plate）

#### 技术选型约束

- **协作教研**与**课堂笔记**应用内所有富文本载体统一采用 `plate`。  
- 首期仅交付 P0、P1 范围，不引入 P2+ 高级能力（如实时协同光标、复杂嵌套块等）。

#### P0：首期基础能力（必做）

- 正文  
- 标题 1、标题 2  
- 基础样式按钮入口  
- 块级插入  
- 块级排序

#### P1：增强格式能力

- 标题层级：H1 / H2 / H3 / H4  
- 列表：有序列表、无序列表、任务列表  
- 引用  
- 文本样式：加粗、斜体、下划线、删除线、代码  
- 链接  
- 表格  
- 对齐方式  
- 文本颜色 / 背景色

#### 模块级能力开关策略（MVP）

- **协作教研（纪要/讨论区等非教案正文区）**：P0 全开；P1 可先关闭「文本颜色 / 背景色」，保证纪要风格统一。  
- **协作教研（教案正文）与课堂笔记**：P0 全开；P1 全开，满足结构化与格式化表达需求。  
- 同一应用内共用一套 `plate` 插件注册机制，差异通过**文档类型/场景**配置开关实现。

#### 验收口径

- P0 验收：从空白文档可完成「标题 + 正文 + 块插入 + 块排序」完整编辑链路。  
- P1 验收：各格式能力支持创建、编辑、保存、回显一致。  
- 数据一致性：保存与再次打开后，H1-H4、列表、表格、链接等核心格式不丢失。


**输入/输出**：

- 输入：编辑器插件配置、模块开关策略、用户编辑行为。  
- 输出：可用能力集合、文档结构化内容、格式化回显结果。

**状态流转**：

- 能力发布状态：P0 已启用 -> P1 已启用 -> P2+ 规划中。  
- 文档编辑状态：未编辑 -> 编辑中 -> 已保存。

**边界情况**：

- 禁用能力在回显时应保持内容可读，不可编辑。  
- 插件加载失败时降级到基础文本编辑能力。  
- 跨模块能力差异需在 UI 中明确提示。

**数据埋点（MVP）**：

- `editor_initialized`：编辑器完成初始化。  
- `editor_block_inserted`：插入块级内容。  
- `editor_format_applied`：应用格式化能力。  
- `editor_save_succeeded`：编辑器内容保存成功。
