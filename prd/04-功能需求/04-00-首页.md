# 04-00-首页

### 6.0 首页（模块化布局）

#### 6.0.1 布局原则

- **禁止**采用传统后台「左侧全局导航树 + 右侧单一工作区」的**左右双栏**主结构。  
- **采用**模块化布局：顶栏、通知栏、应用区（Icon 宫格）、右侧「师说」助手、页脚等区块按栅格/卡片组合排布；主内容区以**模块卡片**为主，可随配置增减。  
- 允许右侧固定「师说」助手面板（可折叠/抽屉），**不属于**上述「传统双栏工作台」信息结构。

#### 6.0.2 顶部菜单栏

| 元素 | 说明 |
| --- | --- |
| 平台名称 | 展示产品名称（与品牌一致）。 |
| 日期 | 展示当前日期 + 星期；**展示格式统一为 `MM.DD.YYYY 星期X`**（中文星期，如「星期三」）。若产品需两位年份简写，可改为 `MM.DD.YY 星期X`，全站顶栏一致即可。 |
| 用户信息 | **姓名**；**角色**；**所属学校**；三者均来自需求方 API（通过登录态关联拉取），本期 C 端仅展示不维护；缺省时使用占位（见边界情况）。 |

#### 6.0.3 通知栏（数据粗统）

- 展示**非精细分析**的汇总信息，例如：  
  - **本周在线时长**（粗粒度，如按登录会话累计或简化规则，具体算法实现阶段定义）；  
  - **待办事项**（与业务状态字段对齐：未提交评课、教研草稿、教案未保存等，规则可与原「工作台」待办一致）。  
- 可扩展其它「一眼即知」的计数/摘要（MVP 以 1～3 条为宜，避免信息过载）。

#### 6.0.4 应用广场（首页内主模块）

- 以 **Icon + 名称**（可加一句话价值/角标）展示多个应用入口，**栅格/卡片模块化排列**，后续通过配置扩展。  
- **MVP 必选入口（三大应用）**：**协作教研**（含教研主题与教案，见 `04-04-协作教研.md`）、**课堂笔记**（见 `04-06-课堂笔记.md`）、**课堂观察**（见 `04-05-课堂观察.md`）。  
- **扩展占位（非 MVP 必选）**：题库、教师成长记录等；未开放时灰显 +「敬请期待」或隐藏（由配置与权限控制）。  
- 点击任一应用 Icon：**进入该应用独立下钻页**（该应用自己的列表/首页，不再停留在首页框架内强嵌业务列表——具体壳层是否保留顶栏由 UI 规范另述，**业务列表必须在应用域内**）。

#### 6.0.5「师说」助手（主 Agent）

- **位置**：页面**右侧**对话区域（可与主内容区并排，支持折叠/宽度拖拽为二期）。  
- **能力**：用户与**主 Agent** 自然语言交互；可**直接描述需求以唤起/跳转单独应用**（与 `04-10-Agent调度.md` 协同，路由规则在实现阶段细化）。  
- **默认提示语**（空会话首条或占位文案）：**你好我是「师说」助手，有问题尽管问。**  
- 与 `04-09-模型对话窗口.md` 中的应用内对话区分：本助手为**首页级、跨应用**入口；进入具体应用后仍以应用内对话为准。

#### 6.0.6 页脚

- 固定展示技术支持文案：**华中师范大学教育大数据应用技术国家工程研究中心 提供技术支持**。

**验收**：登录后默认进入首页；顶栏、通知栏、应用 Icon 区、右侧「师说」助手、页脚均符合上述要求；**协作教研**、**课堂笔记**与**课堂观察**三枚 Icon 可分别进入对应应用下钻页；助手默认提示语正确；整体**非**传统左右双栏工作台主结构。

**输入/输出**：

- 输入：登录态、用户档案、日期/星期、待办与粗统计数据源、应用配置列表、助手会话消息。  
- 输出：首页各模块渲染数据、应用路由跳转、助手回复与跳转指令。

**状态流转**：

- 页面加载：初始化 -> 数据就绪 / 部分降级展示。  
- 助手会话：空会话 -> 对话中 -> 超时/结束；可触发「打开某应用」导航状态。

**边界情况**：

- 用户学校/角色未回填时，顶栏展示「未设置」占位，并提示联系管理员在上游系统补全（本期不在 C 端维护用户资料）。  
- 粗统计接口失败时，通知栏展示占位与重试，不阻断应用区与助手。  
- 扩展应用未开放时不可点击并给出统一提示。  
- 小屏设备：右侧助手默认折叠为入口按钮，展开后覆盖主内容（避免双栏挤压不可用）。

**数据埋点（MVP）**：

- `home_viewed`：进入首页。  
- `home_notification_item_clicked`：点击通知栏某项（若有跳转）。  
- `home_app_icon_clicked`：点击应用 Icon（带 `app_id`）。  
- `shishuo_assistant_opened` / `shishuo_assistant_message_sent`：打开助手 / 发送消息。  
- `shishuo_assistant_navigate_app`：通过助手跳转到某应用（带目标应用标识）。
