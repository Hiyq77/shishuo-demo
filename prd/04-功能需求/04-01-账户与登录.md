# 04-01-账户与登录

### 6.1 账户与登录（首期最小化）

- 首期仅提供最小可用登录能力（如统一入口或单点接入），不在 MVP 中展开注册、找回密码、复杂账号体系细节。  
- 登录后进入 **首页**（模块化落地页，见 `04-00-首页.md`）即可，账号中心与权限精细化策略后续版本补齐。  
- 用户基础信息需满足 **首页顶栏** 展示：姓名、角色、所属学校；业务侧另保留学科、学段等字段，其余字段后置。  
- **本期对接约定**：用户信息由需求方 API 直出并在登录后拉取/回填，C 端仅消费与展示，不提供新增、编辑、删除等维护能力。

**验收**：用户可进入平台并访问 **协作教研**、**课堂观察** 两大核心应用；登录链路不阻断业务流程。


**输入/输出**：

- 输入：登录凭据（或单点票据）、需求方用户信息 API 返回结果。  
- 输出：登录结果、用户会话态、可访问应用范围。

**状态流转**：

- 认证状态：未登录 -> 登录中 -> 已登录/登录失败。  
- 账号状态（MVP）：有效 -> 禁用（占位，后续扩展）。

**边界情况**：

- 单点失败时提示降级登录或稍后重试。  
- 用户基础信息不完整时允许登录，但仅做占位展示并提示联系管理员在上游系统修正（本期 C 端不维护）。  
- 会话过期后跳转登录并保留回跳目标页。

**数据埋点（MVP）**：

- `auth_login_attempted`：发起登录。  
- `auth_login_succeeded`：登录成功。  
- `auth_login_failed`：登录失败。  
- `auth_session_expired`：会话过期。
