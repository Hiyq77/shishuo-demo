# 04-11-资源库

### 6.11 资源沉淀与检索（MVP）

#### 6.11.1 定位与入口

- **定位**：承接 **协作教研**、**课堂观察** 等应用产出的「可检索资产」，支持列表浏览、详情预览、回跳源业务对象。  
- **入口（MVP）**：  
  - 各应用内「入库成功」后的跳转入口（toast/按钮）；  
  - 首页扩展位或顶栏入口（若本期不做独立导航入口，则**至少**保证应用内可进入资源库列表，并在 `08-指标与验收.md` 联合验收）。

#### 6.11.2 资源类型（MVP）

| 类型 | 说明 | 典型来源 |
| --- | --- | --- |
| 教研纪要 | 结构化纪要正文 | 协作教研主题 |
| 教案 | 定稿/草稿版本（以入库时版本为准） | 协作教研 |
| 课堂观察报告 | 结构化报告正文 | 课堂观察 |

#### 6.11.3 列表与详情

- **列表字段**：标题、类型、来源应用、来源对象 ID、创建人、更新时间。  
- **筛选（MVP）**：按类型、按时间范围（可选其一或并存，实现前定稿）。  
- **搜索（MVP）**：标题关键词。  
- **详情**：正文预览、复制全文、导出（MVP 可先 **Markdown/纯文本**）；提供「打开来源」跳转至协作教研/课堂观察对应详情页（至少一种回源方式）。

#### 6.11.4 入库与状态流转

- **入库状态**：未入库 → 入库中 → 入库成功 / 入库失败。  
- **失败处理**：入库失败时**不丢弃**已生成正文；列表或详情展示失败原因占位，并提供 **重试入库**（幂等）。  
- **来源绑定**：每条资源必须携带 `source_app` + `source_object_id`，用于回跳与审计。

#### 6.11.5 边界情况

- 来源对象已删除或不可达时：资源仍可读，回源按钮置灰并提示「来源不可用」。  
- 标题重复：允许并存，以更新时间与来源 ID 区分展示。  
- 导出失败：提示重试，不影响详情阅读。

#### 6.11.6 数据埋点（MVP）

- `repo_list_viewed`：进入资源库列表。  
- `repo_item_opened`：打开资源详情。  
- `repo_source_opened`：回跳来源对象。  
- `repo_export_triggered`：触发导出。  
- `repo_ingest_retry_clicked`：点击重试入库。

**验收**：三类资源均可从应用完成入库后在资源库中检索、打开详情、回跳来源对象至少一种路径可用；入库失败可重试且不丢正文。
