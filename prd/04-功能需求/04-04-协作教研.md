# 04-04-协作教研

### 6.4 协作教研（统一命名）

> **命名约定（全 PRD 对齐）**：**「协作教研」即本应用唯一名称**，其能力范围 **等价于原独立模块「教案生成」**（分步向导、富文本教案、改写、入库等）。  
> PRD 中不再单独出现「教案生成」「教研空间」作为产品模块名；历史表述一律映射为本节。课堂笔记已独立为应用，见 `04-06-课堂笔记.md`。

#### 6.4.1 教研主题列表

- 字段：主题、组织类型（年级组/课题组/学科组）、学科、学段、状态（进行中/已归档）、发起人、更新时间。  
- 筛选：学科、状态。  
- 操作：新建教研主题。

**输入/输出**：

- 输入：筛选条件（学科、状态）、分页参数、新建动作。  
- 输出：教研主题列表、状态统计（可选）、可跳转详情 ID。

**状态流转**：

- 主题状态：进行中 → 已归档。  
- 纪要状态：未生成 → 已生成 → 已入库（可重生成覆盖内容）。

**边界情况**：

- 无数据时展示空状态与「新建教研」引导。  
- 筛选无结果时保留筛选条件并提示清空。  
- 用户无创建权限时仅展示只读列表（若后续引入细粒度权限）。

**数据埋点（MVP）**：

- `research_list_viewed`：进入列表页。  
- `research_filter_applied`：应用筛选条件。  
- `research_create_clicked`：点击新建教研主题。

#### 6.4.2 教研主题详情（核心页面）

- **基础信息**：主题、组织类型（年级组/课题组/学科组）、学科、学段、标签（可选）、状态。  
- **背景与目标**：富文本。  
- **讨论区**：评论/回复（MVP 单层即可）。  
- **资料区**：链接 + 备注；文件上传（若成本高可二期，PRD 建议 MVP **至少支持链接**）。  
- **共创与共享**：支持在**协作教研**应用内共创教案，并支持分享课程分析结果（与课堂观察模块衔接处由链接/关联实现）。  
- **AI 能力（固定）**：  
  - 生成讨论提纲  
  - 从讨论生成纪要（输入：讨论内容；输出：结构化纪要）  
  - 提炼争议点与行动项  
- **产出**：一键保存「教研纪要」到资源库（类型=教研纪要），并记录来源主题 ID。  
- **成员（弱）**：展示参与者列表；邀请（可用邮箱占位，二期完善）。

**输入/输出**：

- 输入：主题基础信息、讨论文本、资料链接、成员信息、AI 指令。  
- 输出：结构化纪要（标题/要点/争议点/行动项）、资源库纪要条目、来源关联关系。

**状态流转**：

- 详情页对象：草稿中 → 进行中 → 已归档。  
- AI 任务：待执行 → 生成中 → 成功/失败。  
- 入库任务：未入库 → 入库成功/入库失败（可重试）。

**边界情况**：

- 讨论内容过少时，限制纪要生成并提示补充输入。  
- AI 生成失败时保留输入上下文，支持一键重试。  
- 入库失败时不丢失纪要正文，允许稍后再次入库。  
- 主题归档后默认不可新增讨论（可只读回看）。

**数据埋点（MVP）**：

- `research_detail_viewed`：进入教研主题详情。  
- `research_outline_generated`：生成讨论提纲。  
- `research_minutes_generated`：生成纪要。  
- `research_action_items_generated`：提炼行动项。  
- `research_minutes_saved_to_repo`：纪要入库成功。  
- `research_minutes_save_failed`：纪要入库失败。

**验收**：能完成一次完整教研闭环：创建主题 → 讨论 → AI 纪要 → 入库。

#### 6.4.3 教案列表与新建/编辑（分步向导）

> 本节为原「教案生成」全部能力，并入协作教研统一交付。

##### 教案列表（建议纳入 MVP）

- 字段：标题、学科、学段、状态（草稿/已定稿）、更新时间。  
- 操作：新建、进入编辑。

**输入/输出**：

- 输入：筛选条件（学科、学段、状态）、分页参数、新建动作。  
- 输出：教案列表、草稿/定稿状态、可跳转编辑对象 ID。

**状态流转**：

- 文档状态：草稿 → 已定稿（定稿后可再次编辑并回到草稿）。  
- 版本状态（若开启）：当前版本 → 新版本（保留历史版本）。

**边界情况**：

- 无教案时展示空状态与「新建教案」引导。  
- 列表加载失败时支持重试，且不影响已缓存内容浏览。  
- 用户仅有查看权限时隐藏新建与编辑入口（按权限策略生效）。

**数据埋点（MVP）**：

- `lesson_plan_list_viewed`：进入教案列表。  
- `lesson_plan_create_clicked`：点击新建教案。  
- `lesson_plan_opened`：打开教案详情/编辑页。

##### 教案新建/编辑（分步向导）

- **步骤 1 基础信息**：学科、学段、教材版本（文本或枚举）、课时长度、课题名称。  
- **步骤 2 教学目标**：教学目标、重难点、学情简述。  
- **步骤 3 AI 生成**：生成初稿；支持再次生成（覆盖或新版本，二选一，建议 **新版本**）。  
- **步骤 4 富文本编辑与改写**：  
  - 载体采用 `plate` 富文本编辑器，支持插入多种 block（按 `04-08-编辑器能力与分期.md` 分期能力）。  
  - 结构化板块：导入、新授、活动、小结、作业、板书、反思（可按基教习惯微调命名）。  
  - **改写按钮（至少 3 个）**：缩短时长、增加学生活动、公开课/展示风格（文案可调整）。  
- **步骤 5 多人共创**：支持多人参与同一教案共创，满足联合教案业务场景（首期可采用非实时协同策略）。  
- **保存**：草稿自动保存（间隔保存或手动保存，至少手动保存）。  
- **入库**：保存到资源库（类型=教案），记录版本号（若实现版本）。

**输入/输出**：

- 输入：基础信息、教学目标、AI 指令、编辑器内容、改写指令、共创编辑操作。  
- 输出：教案初稿/改写结果、结构化富文本内容、版本记录、资源库教案条目。

**状态流转**：

- 生成状态：未生成 → 生成中 → 成功/失败。  
- 编辑状态：未保存改动 → 已保存草稿 → 已定稿。  
- 入库状态：未入库 → 入库成功/入库失败（失败可重试）。

**边界情况**：

- 必填信息缺失时不可触发 AI 生成，并提示缺失字段。  
- AI 生成失败时保留当前输入并支持重试或切换指令。  
- 共创冲突时按最后保存版本提示冲突并提供人工合并。  
- 定稿后再编辑需二次确认，避免误改已发布内容。

**数据埋点（MVP）**：

- `lesson_plan_generate_started`：触发初稿生成。  
- `lesson_plan_generate_succeeded`：初稿生成成功。  
- `lesson_plan_rewrite_clicked`：触发改写。  
- `lesson_plan_saved_draft`：保存草稿。  
- `lesson_plan_published`：标记为定稿。  
- `lesson_plan_saved_to_repo`：教案入库成功。  
- `lesson_plan_save_failed`：教案入库失败。

**验收**：能生成并编辑富文本教案；支持 block 插入与多人共创；改写与入库可用。

#### 6.4.4 与课堂笔记应用的联动

- **定位说明**：课堂笔记已从协作教研内嵌能力升级为独立应用，完整需求见 `04-06-课堂笔记.md`。  
- **联动方式**：协作教研主题详情与教案编辑场景可一键跳转课堂笔记，并带入当前主题/教案上下文（如主题 ID、学科、学段）。  
- **产出回流**：课堂笔记内沉淀的结构化内容可回写为协作教研主题素材或教案草稿片段，减少重复录入。  
- **权限继承**：从协作教研进入课堂笔记时，协作范围与可见性遵循来源对象权限（私有/协作）。

**输入/输出**：

- 输入：来源对象 ID（教研主题/教案）、跳转动作、上下文参数、回写动作。  
- 输出：课堂笔记打开态、上下文笔记草稿、回流后的主题素材或教案片段。

**状态流转**：

- 联动状态：未联动 → 已跳转课堂笔记 → 已回流协作教研。  
- 回流状态：未回流 → 回流成功/回流失败（支持重试）。

**边界情况**：

- 来源对象不存在或无权限时，阻止跳转并给出明确提示。  
- 回流失败时保留课堂笔记内容，不影响笔记继续编辑。  
- 回流目标已变更（如教案已归档）时，提示用户选择新目标或仅保存到资源库。

**数据埋点（MVP）**：

- `research_note_app_navigated`：从协作教研跳转课堂笔记。  
- `research_note_context_bound`：课堂笔记成功绑定协作上下文。  
- `research_note_content_backfilled`：课堂笔记内容回流成功。  
- `research_note_content_backfill_failed`：课堂笔记内容回流失败。

**验收**：协作教研可稳定跳转课堂笔记并完成上下文带入；课堂笔记产出可回流到主题素材或教案内容，形成可追踪链路。
