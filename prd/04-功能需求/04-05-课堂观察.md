# 04-05-课堂观察

### 6.5 课堂观察

#### 6.5.1 课堂列表页

- **视图模式**：支持「视频封面卡片流」与「日历视图」两种模式；默认进入「视频封面卡片流」。  
- **卡片信息（MVP 必显）**：课程封面（视频第一帧）、学科标签、授课教师、分析状态（成功 / 分析中 / 失败）。  
- **筛选与检索**：学科、授课教师、时间范围、分析状态；支持关键词检索课题。  
- **列表操作**：新建上传、进入课堂详情、失败任务重试。

**输入/输出**：

- 输入：筛选条件、视图模式切换、分页参数、重试动作。  
- 输出：课堂视频列表对象、分析状态集合、可跳转详情对象 ID。

**状态流转**：

- 分析状态：待分析 -> 分析中 -> 成功 / 失败。  
- 展示状态：卡片流 <-> 日历视图（仅展示模式切换，不改变业务状态）。

**边界情况**：

- 无数据时展示空状态与「上传课堂视频」引导。  
- 日历所选日期无数据时，提示切换日期或回到卡片流查看全部。  
- 列表加载失败时支持重试，不阻断上传入口。  
- 分析失败卡片需给出失败标记与「重新分析」操作。

**数据埋点（MVP）**：

- `classroom_observation_list_viewed`：进入课堂观察列表。  
- `classroom_observation_view_switched`：切换卡片流 / 日历视图。  
- `classroom_observation_filter_applied`：应用筛选条件。  
- `classroom_video_retry_clicked`：点击失败任务重试。

#### 6.5.2 视频手动上传

- **入口**：课堂列表页点击「上传课堂视频」后，页面右侧滑出 drawer。  
- **必填字段**：  
  - **课程名称**：通过筛选器选择基教学科（MVP 至少包含 **语文、数学、英语**；其余学科可按配置扩展，与学段课表口径一致即可）。  
  - **课堂名称**：手动输入（短文本，具体长度校验由实现定义，建议 1～80 字）。  
  - **授课日期**：年月日选择器。  
  - **课堂时间**：起止时间选择器（同一自然日内的开始/结束时刻；跨日规则若不支持则校验禁止跨日）。  
  - **视频文件**：上传载体，格式与大小限制见字段校验。  
- **选填字段**：学段、授课教师、评课人、备注、关联教案（与协作教研教案对象可选关联）。  
- **上传反馈**：展示上传进度、文件时长与大小校验结果、分析排队状态。

**输入/输出**：

- 输入：视频文件、课例元数据、提交动作（保存并分析 / 仅保存草稿）。  
- 输出：课堂视频对象 ID、上传结果、分析任务 ID。

**状态流转**：

- 上传状态：待上传 -> 上传中 -> 上传成功 / 上传失败 / 已取消。  
- 任务状态：草稿 -> 已提交分析 -> 分析中 -> 成功 / 失败。

**字段校验**：

- 缺少必填字段时不可提交，并定位到具体字段。  
- 视频格式不支持或超出大小限制时，阻止上传并提示可接受范围。  
- 重名视频允许上传，但需提示“将作为新记录保存”。

**边界情况**：

- 上传中关闭 drawer 时需二次确认，避免误中断。  
- 网络抖动导致失败时允许重试，保留已填写元数据。  
- 上传成功但分析任务创建失败时，记录保留为草稿并可手动重试发起分析。

**数据埋点（MVP）**：

- `classroom_video_upload_drawer_opened`：打开上传 drawer。  
- `classroom_video_upload_submitted`：提交上传。  
- `classroom_video_upload_succeeded`：上传成功。  
- `classroom_video_upload_failed`：上传失败。  
- `classroom_analysis_task_created`：成功创建分析任务。

#### 6.5.3 报告交互（视频-报告-AI 三位一体）

- **页面结构**：左侧视频播放器，中部结构化报告，右侧常驻 AI 助手入口（可折叠）。  
- **报告内容**：课堂概览、关键片段、互动统计、问题与建议；支持保存为评课报告并入库。  
- **AI 能力**：支持围绕当前课堂视频与报告进行多轮追问、解释、归因与改进建议生成。  
- **交互闭环**：AI 回复可回填到报告备注区，报告关键段可一键发起追问。

**循证定位（时间戳锚点机制）**：

- AI 回答中必须携带至少一个可点击时间戳锚点（如 `15:20`、`18:40`）。  
- 点击锚点后，左侧播放器立即执行 seek 并自动播放对应片段。  
- 多个锚点按时间顺序展示，支持连续点击定位。  
- 若 AI 回答未生成锚点，系统需提示并触发补充生成，不可直接以纯文本结论结束。

**追问机制**：

- AI 给出结论后，消息下方自动生成推荐问题（如「查看这段时间的提问明细」）。  
- 推荐问题支持一键发送并继承当前锚点上下文。  
- 用户也可编辑推荐问题后再发送，保持多轮上下文连续。

**输入/输出**：

- 输入：视频分析结果、结构化报告内容、用户问题、上下文锚点。  
- 输出：带锚点的 AI 回答、推荐追问、可回填报告片段、入库报告对象。

**状态流转**：

- 报告状态：未生成 -> 生成中 -> 已生成 -> 已入库 / 入库失败。  
- 对话状态：未开始 -> 对话中 -> 完成 / 失败。  
- 锚点联动状态：未定位 -> 已定位播放 -> 已回到报告上下文。

**边界情况**：

- AI 对话失败时保留已有报告内容与会话记录，支持重试。  
- 视频不可播放时，锚点点击需提示错误并提供刷新播放器入口。  
- 报告入库失败时不丢失生成结果，允许后续再次入库。  
- 无分析结果时仅允许上传与基础信息查看，不开放报告问答入口。

**数据埋点（MVP）**：

- `classroom_report_viewed`：进入报告交互页。  
- `classroom_ai_chat_opened`：打开课堂观察 AI 助手。  
- `classroom_ai_anchor_rendered`：AI 回答生成时间戳锚点。  
- `classroom_ai_anchor_clicked`：点击时间戳锚点触发视频定位。  
- `classroom_ai_followup_clicked`：点击推荐追问。  
- `classroom_report_saved_to_repo`：评课报告入库成功。  
- `classroom_report_save_failed`：评课报告入库失败。

**验收**：

- 列表默认展示视频封面卡片流，并可切换日历视图。  
- 手动上传通过右侧 drawer 完成，提交后可见上传与分析状态。  
- 报告页可实现视频-报告-AI 联动：AI 回答包含可点击时间戳，点击后视频可立即跳转播放。  
- AI 结论后可自动生成推荐追问并支持一键继续对话。  
- 报告可保存入库并可追溯来源课堂视频对象。
