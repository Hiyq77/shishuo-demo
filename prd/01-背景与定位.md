# 01 背景与定位

## 1. 背景与问题

### 1.1 背景

- 学校与区域教研强调**协作备课、听评课、资源沉淀**，但过程分散在会议、文档与即时通讯中，难形成可检索、可复用的资产。  
- 教师备课、上课、评课、反思之间**缺少统一载体**，优质经验难以闭环回到下一轮备课。  
- 生成式 AI 已具备**教案起草、纪要总结、评课报告结构化输出**等能力，但需要嵌入**可信工作流**（权限、版本、校本资源），避免「一次性对话」无法落地。

### 1.2 要解决的问题（MVP 聚焦）

1. **教研协作低效**：讨论过程难沉淀，纪要依赖人工，行动项易丢失。
2. **备课产出不稳定**：教案质量依赖个人经验，格式与环节不完整，难复用。
3. **听评课成本高**：记录碎片化，评价维度不一致，反馈难结构化、难追踪改进。
4. **资产难复用**：教案、纪要、评课报告散落，无法统一检索与关联。

---

## 2. 产品定位与包装（双叙事合一）

### 2.1 一句话定位

面向基教场景的 **「AI 教研协作 + AI 教学备课 + AI 听评课」一体化平台**，以**模块化应用**呈现能力，以**资源库**沉淀成果，形成**「备课 → 上课 → 评价 → 反思 → 再备课」**的教学改进闭环。

### 2.2 对外两条主线（同一产品、不同采购话术）


| 包装方向           | 对外价值主张                            | MVP 中对应落地                       |
| -------------- | --------------------------------- | ------------------------------- |
| **AI 教研平台**    | 提升区域/校本教研效率：主题协作、资料汇聚、纪要沉淀、成果入库   | **协作教研** + 资源库 + 教研类 AI         |
| **AI 教学全链路平台** | 贯穿教学关键节点：协作备课与教案迭代、课堂证据与评价、改进建议回流备课 | **协作教研** + **课堂观察** + 资源关联（弱闭环） |


### 2.3 MVP 边界说明

- MVP 强调**流程跑通、资产沉淀、两大应用可用**（**协作教研**、**课堂观察**），不追求复杂数据分析与多租户极致能力。  
- 「全链路」在 MVP 中体现为：**对象可关联 + 统一资源库 + 首页聚合**（待办与粗统计等在首页通知栏呈现），而非全自动数据中台。

---

## 3. 目标与非目标

### 3.1 目标（MVP）

1. 支持教师完成 **协作教研** 端到端：**教研主题与讨论 → 纪要并入库**；**按向导生成教案 → 多轮改写 → 定稿/草稿管理 → 入库**；**课堂笔记** 的记录与关联（详见 `04-功能需求/04-04-协作教研.md`）。
2. 支持教师/组长完成 **课堂观察** 端到端：**创建听评课或自评任务 → 记录与量表 → AI 报告 → 入库**。
3. 提供 **首页** 聚合待办与粗统计（通知栏）、**应用广场 Icon** 作为模块化入口（**协作教研**、**课堂观察**，可扩展）。
4. 提供 **资源库**，支持按类型检索与从应用跳转回源对象（至少一种关联方式）。

### 3.2 非目标（MVP 不做）

- 不替代 LMS 的全流程教学管理（排课、成绩、学籍等）。  
- 不做复杂课堂实录 AI（实时转写、多模态课堂分析）——可预留入口，默认不做。  
- 不做区域级大数据看板与强治理报表（仅保留基础计数/列表级「弱数据」）。  
- 不做完整题库、作业批改、学情大数据预测。
