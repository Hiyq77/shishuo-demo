# 《基教 AI 教学教研平台》产品需求文档（PRD）

**文档版本**：v1.0（MVP）  
**适用阶段**：义务教育 / 普通高中（统称「基教」）  
**产品代号**：待定  
**文档日期**：2026-04-17  

---

## 1. 背景与问题

### 1.1 背景

- 学校与区域教研强调**协作备课、听评课、资源沉淀**，但过程分散在会议、文档与即时通讯中，难形成可检索、可复用的资产。  
- 教师备课、上课、评课、反思之间**缺少统一载体**，优质经验难以闭环回到下一轮备课。  
- 生成式 AI 已具备**教案起草、纪要总结、评课报告结构化输出**等能力，但需要嵌入**可信工作流**（权限、版本、校本资源），避免「一次性对话」无法落地。

### 1.2 要解决的问题（MVP 聚焦）

1. **教研协作低效**：讨论过程难沉淀，纪要依赖人工，行动项易丢失。
2. **备课产出不稳定**：教案质量依赖个人经验，格式与环节不完整，难复用。
3. **听评课成本高**：记录碎片化，评价维度不一致，反馈难结构化、难追踪改进。
4. **资产难复用**：教案、纪要、评课报告散落，无法统一检索与关联。

---

## 2. 产品定位与包装（双叙事合一）

### 2.1 一句话定位

面向基教场景的 **「AI 教研协作 + AI 教学备课 + AI 听评课」一体化平台**，以**模块化应用**呈现能力，以**资源库**沉淀成果，形成**「备课 → 上课 → 评价 → 反思 → 再备课」**的教学改进闭环。

### 2.2 对外两条主线（同一产品、不同采购话术）


| 包装方向           | 对外价值主张                            | MVP 中对应落地                       |
| -------------- | --------------------------------- | ------------------------------- |
| **AI 教研平台**    | 提升区域/校本教研效率：主题协作、资料汇聚、纪要沉淀、成果入库   | **协作教研** + 资源库 + 教研类 AI         |
| **AI 教学全链路平台** | 贯穿教学关键节点：教案生成与迭代、课堂证据与评价、改进建议回流备课 | **教案生成** + **课堂评价** + 资源关联（弱闭环） |


### 2.3 MVP 边界说明

- MVP 强调**流程跑通、资产沉淀、三应用可用**，不追求复杂数据分析与多租户极致能力。  
- 「全链路」在 MVP 中体现为：**对象可关联 + 统一资源库 + 工作台聚合**，而非全自动数据中台。

## 3. 目标与非目标

### 3.1 目标（MVP）

1. 支持教师完成：**发起/参与教研 → 生成纪要并入库**。
2. 支持教师完成：**按向导生成教案 → 多轮改写 → 定稿/草稿管理 → 入库**。
3. 支持教师/组长完成：**创建听评课任务 → 记录与量表 → AI 报告 → 入库**。
4. 提供 **工作台** 聚合待办与最近使用；提供 **应用广场** 作为模块化入口。
5. 提供 **资源库**，支持按类型检索与从应用跳转回源对象（至少一种关联方式）。

### 3.2 非目标（MVP 不做）

- 不替代 LMS 的全流程教学管理（排课、成绩、学籍等）。  
- 不做复杂课堂实录 AI（实时转写、多模态课堂分析）——可预留入口，默认不做。  
- 不做区域级大数据看板与强治理报表（仅保留基础计数/列表级「弱数据」）。  
- 不做完整题库、作业批改、学情大数据预测。

## 4. 用户与角色

### 4.1 角色（MVP）


| 角色              | 核心诉求                   |
| --------------- | ---------------------- |
| **任课教师**        | 写教案、参与教研、被评课或自评、管理个人资源 |
| **备课组长 / 教研组长** | 发起教研、汇总成果、组织听评课（若权限允许） |
| **校管理者（可选占位）**  | 查看有限列表与汇总（MVP 可极弱）     |


### 4.2 权限（MVP 简化）

- 默认 **同校可见**（若仅单校部署，可先不做跨校）。  
- 资源可见性：**仅本人 / 校内公开（占位）** 二选一或简化为「创建人可编辑，同角色只读」——需在实现前定一条默认策略（PRD 建议：**默认本人，可显式分享到教研主题参与者**）。

## 6. 功能需求（按模块）

### 6.1 账户与登录（首期最小化）

- 首期仅提供最小可用登录能力（如统一入口或单点接入），不在 MVP 中展开注册、找回密码、复杂账号体系细节。  
- 登录后进入工作台即可，账号中心与权限精细化策略后续版本补齐。  
- 用户基础信息仅保留业务必需字段（姓名、学科、学段），其余字段后置。

**验收**：用户可进入平台并访问三大核心应用；登录链路不阻断业务流程。

---

### 6.2 工作台

**目的**：降低冷启动成本，强化「全链路」感知。

**模块**：

- **快捷入口**：发起教研、新建教案、新建评课。  
- **待办（可规则化）**：未完成的教研草稿、未提交的评课、未保存的教案（依赖状态字段）。  
- **最近使用**：最近打开的教研主题、教案、评课单（时间倒序，上限 10）。  
- **弱数据概览（可选）**：本周新建教案数、完成评课数（计数即可）。

**验收**：三类对象可从最近记录一键进入；快捷入口可用。

---

### 6.3 应用广场

- 展示 3 个应用卡片：**协作教研、教案生成、课堂评价**。  
- 每张卡片：名称、一句话价值、进入按钮、上次使用时间（有则显示）。

**验收**：点击进入对应应用首页/列表。

---

### 6.4 协作教研

#### 6.4.1 教研列表

- 字段：主题、组织类型（年级组/课题组/学科组）、学科、学段、状态（进行中/已归档）、发起人、更新时间。  
- 筛选：学科、状态。  
- 操作：新建教研。

#### 6.4.2 教研详情（核心页面）

- **基础信息**：主题、组织类型（年级组/课题组/学科组）、学科、学段、标签（可选）、状态。  
- **背景与目标**：富文本。  
- **讨论区**：评论/回复（MVP 单层即可）。  
- **资料区**：链接 + 备注；文件上传（若成本高可二期，PRD 建议 MVP **至少支持链接**）。  
- **共创与共享**：支持在教研空间内共创教案，并支持分享课程分析结果。  
- **AI 能力（固定）**：  
  - 生成讨论提纲  
  - 从讨论生成纪要（输入：讨论内容；输出：结构化纪要）  
  - 提炼争议点与行动项
- **产出**：一键保存「教研纪要」到资源库（类型=教研纪要），并记录来源教研 ID。  
- **成员（弱）**：展示参与者列表；邀请（可用邮箱占位，二期完善）。

**验收**：能完成一次完整教研闭环：创建 → 讨论 → AI 纪要 → 入库。

---

### 6.5 教案生成

#### 6.5.1 教案列表（建议纳入 MVP）

- 字段：标题、学科、学段、状态（草稿/已定稿）、更新时间。  
- 操作：新建、进入编辑。

#### 6.5.2 教案新建/编辑（分步向导）

- **步骤 1 基础信息**：学科、学段、教材版本（文本或枚举）、课时长度、课题名称。  
- **步骤 2 教学目标**：教学目标、重难点、学情简述。  
- **步骤 3 AI 生成**：生成初稿；支持再次生成（覆盖或新版本，二选一，建议 **新版本**）。  
- **步骤 4 富文本编辑与改写**：  
  - 载体采用 `plate` 富文本编辑器，支持插入多种 block（按 6.11 分期能力）。  
  - 结构化板块：导入、新授、活动、小结、作业、板书、反思（可按基教习惯微调命名）。  
  - **改写按钮（至少 3 个）**：缩短时长、增加学生活动、公开课/展示风格（文案可调整）。
- **步骤 5 多人共创**：支持多人参与同一教案共创，满足联合教案业务场景（首期可采用非实时协同策略）。
- **保存**：草稿自动保存（间隔保存或手动保存，至少手动保存）。  
- **入库**：保存到资源库（类型=教案），记录版本号（若实现版本）。

**验收**：能生成并编辑富文本教案；支持 block 插入与多人共创；改写与入库可用。

---

### 6.6 课堂评价

#### 6.6.1 评课列表

- 字段：课题、学科、授课教师、评课人、日期、状态（草稿/已提交）。  
- 筛选：学科、时间范围、状态。

#### 6.6.2 新建评课

- **课例信息**：学科、学段、课题、日期、授课教师、评课人（默认当前用户）。  
- **模式**：听评课 / 自评。  
- **视频输入**：支持基于课堂视频发起评价任务。  
- **量表**：内置 **1 套基教通用课堂观察量表**（固定维度与评分制，如 1–5 分）。  
- **证据输入**：课堂要点文字记录（必填其一：记录或量表打分）。

#### 6.6.3 评课详情

- **量表打分**：各维度分数 + 简短文字说明。  
- **课堂记录**：自由文本。  
- **视频评价结果**：支持承接现有成熟课堂视频评价流程产出的分析结果（平台首期以融合与承接为主）。  
- **AI 能力**：  
  - 生成简评  
  - 生成改进建议  
  - 生成结构化评课报告（标题、亮点、问题、建议、下一步）  
  - 支持通过模型对话进一步追问与解释报告内容
- **入库**：保存评课报告到资源库（类型=评课报告），关联评课单 ID。

**验收**：能基于课堂视频完成评价并产出报告；用户可通过模型对话深入理解报告；结果可入库并可追溯来源流程。

---

### 6.7 知识库

- **类型**：教研纪要、教案、评课报告（三类必选）。  
- **列表**：标题、类型、学科、时间、来源应用（可点击跳转）。  
- **详情**：预览正文、复制全文、下载（MVP 可先 **导出 Markdown/纯文本**，PDF 可二期）。  
- **搜索**：标题关键词。

**验收**：三类产出均可检索与打开；能回到源对象（至少通过 ID 跳转详情页）。

---

### 6.8 班级与学科（轻量）

- 维护「常用班级」列表：班级名、年级（可选）。  
- 在教案/评课表单中可下拉选择（非强制）。

**验收**：班级可增删改；表单可选用。

---

### 6.9 设置

- 学科、学段默认值。  
- 通知开关（占位即可）。

---

### 6.10 帮助与合规

- 简要说明三大应用怎么用。  
- 隐私说明：用户输入与生成内容用途、是否用于模型训练（按实际政策填写，MVP 至少要有文案占位）。

---

### 6.11 编辑器能力范围与分期（Plate）

#### 技术选型约束

- `教案生成` 与 `协作教研` 的文档载体统一采用 `plate`。  
- 首期仅交付 P0、P1 范围，不引入 P2+ 高级能力（如实时协同光标、复杂嵌套块等）。

#### P0：首期基础能力（必做）

- 正文  
- 标题 1、标题 2  
- 基础样式按钮入口  
- 块级插入  
- 块级排序

#### P1：增强格式能力

- 标题层级：H1 / H2 / H3 / H4  
- 列表：有序列表、无序列表、任务列表  
- 引用  
- 文本样式：加粗、斜体、下划线、删除线、代码  
- 链接  
- 表格  
- 对齐方式  
- 文本颜色 / 背景色

#### 模块级能力开关策略（MVP）

- `协作教研`：P0 全开；P1 可先关闭「文本颜色 / 背景色」，保证教研纪要风格统一。  
- `教案生成`：P0 全开；P1 全开，满足教案结构化与格式化表达需求。  
- 两模块共用同一套 `plate` 插件注册机制，差异通过配置开关实现。

#### 验收口径

- P0 验收：从空白文档可完成「标题 + 正文 + 块插入 + 块排序」完整编辑链路。  
- P1 验收：各格式能力支持创建、编辑、保存、回显一致。  
- 数据一致性：保存与再次打开后，H1-H4、列表、表格、链接等核心格式不丢失。

---

### 6.12 应用内模型对话窗口

#### 目标

- 各应用均提供统一的模型对话窗口，支持用户通过自然语言直接提出任务并获得可落地结果。  
- 对话能力与现有按钮式 AI 能力并存，用户可自由选择「指令按钮」或「自由对话」。

#### 覆盖范围（MVP）

- `协作教研`：支持对当前教研主题进行提纲生成、纪要整理、行动项拆解、讨论改写。  
- `教案生成`：支持教案初稿生成、分段改写、风格调整、补充活动与作业。  
- `课堂评价`：支持课堂记录总结、简评生成、改进建议、结构化评课报告草拟。

#### 交互与入口

- 每个应用详情页右侧提供固定「AI 对话窗口」入口（可折叠）。  
- 支持多轮对话，保留当前页面会话上下文。  
- 支持快捷指令（如「帮我总结」「帮我改写」）一键填充到输入框。

#### 上下文注入（MVP 必做）

- 对话调用需自动携带当前页面核心上下文（如主题信息、教案内容、评课记录），减少用户重复输入。  
- 明确显示「本次对话已引用的上下文范围」（如：当前文档、当前量表记录）。  
- 用户可手动补充输入，但 MVP 不做复杂多文档上下文拼接。

#### 结果回填与保存

- 对话结果支持「插入当前文档」与「替换选中内容」。  
- 对话结果支持「保存为草稿版本」或「追加到当前对象」。  
- 所有回填操作需可撤销，避免误覆盖。

#### 验收口径

- 三个应用均可打开并使用对话窗口完成至少 1 个核心任务。  
- 对话结果可回填到当前对象且保存后可回显。  
- 上下文注入可生效，且用户可感知当前引用范围。  
- 对话失败或超时时有明确错误提示与重试入口。

---

### 6.13 智能识别与 Agent 调度

#### 目标

- 对用户自然语言输入进行任务意图识别，自动选择更合适的处理方式（对话问答、文稿编辑、结构化生成）。  
- 在可识别场景下自动调度专职 Agent，提升结果质量与任务完成效率。

#### 识别与路由规则（MVP）

- 系统需对输入进行基础任务分类：  
  - `对话模式`：讨论教学思路、追问细节、即时答疑  
  - `文稿编辑`：围绕当前文档补写、润色、改写、整理  
  - `结构化生成`：按固定模板输出（如教案、评课报告、练习题）  
- 当识别置信度高于阈值时，自动执行路由；低于阈值时，提示用户选择处理方式。  
- 路由结果需在 UI 中可感知（显示当前已调用的能力或 Agent）。

#### Agent 能力清单（MVP）

- 教案 Agent：负责教案生成、分段改写、教学活动与作业补充。  
- 导图 Agent：负责将主题或文稿转为结构化导图大纲。  
- 图谱 Agent：负责知识点关系抽取与关系图谱文本化输出。  
- 课件 Agent：负责课件页纲要、讲授节奏与页面文案草案。  
- 练习 Agent：负责分层练习题生成与参考答案/解析草案。  
- 评价 Agent：负责课堂评价总结、改进建议与评课报告结构化输出。  
- 知识库 Agent：负责知识库检索增强、要点引用与来源整合摘要。

#### 调度与回退机制

- 默认由统一调度层按任务类型分发到对应 Agent。  
- 若目标 Agent 不可用或执行失败，自动回退到通用模型并提示用户。  
- 用户可手动切换 Agent 或改用通用对话模式。  
- 所有调度请求需记录任务类型、路由结果、耗时与状态，用于后续优化。

#### 验收口径

- 至少 3 类任务类型（对话、文稿编辑、结构化生成）可自动识别并正确路由。  
- 7 类 Agent 均有至少 1 个可用场景，且输出可回填到当前业务对象。  
- 置信度低场景下可触发用户确认，不出现静默错误路由。  
- Agent 失败后可回退到通用模型，流程不中断。

## 7. 「全链路」闭环（MVP 可落地的关联）


| 关联      | MVP 要求                   |
| ------- | ------------------------ |
| 教案 ↔ 教研 | 教案可选「关联教研主题」（下拉选择进行中的教研） |
| 评课 ↔ 教案 | 评课表单可选「关联教案」（若存在）        |
| 统一沉淀    | 所有产出进资源库，带类型与来源 ID       |


**说明**：MVP 不强制做自动「改进建议回流教案」，但 UI 可提供「从评课打开关联教案」以完成人工闭环。

## 8. AI 需求（统一原则）

### 8.1 能力清单（MVP）

- 应用内模型对话窗口：协作教研、教案生成、课堂评价均支持多轮自然语言对话  
- 智能识别与任务路由：支持自动识别「对话模式 / 文稿编辑 / 结构化生成」并选择处理方式  
- Agent 调度：支持教案 Agent、导图 Agent、图谱 Agent、课件 Agent、练习 Agent、评价 Agent、知识库 Agent  
- 教研：提纲、纪要、争议点/行动项  
- 教案：初稿、改写（多指令）  
- 评课：简评、改进建议、结构化报告

### 8.2 质量与安全原则

- 输出需**结构化**（标题、列表），便于入库与展示。  
- 明确提示：AI 生成内容需教师审核后使用。  
- 支持「重新生成」与「人工编辑优先」。

### 8.3 模型与数据（PRD 层）

- 记录：调用类型、对象 ID、用户 ID、时间（用于排错与计费预留）。  
- 训练使用：默认 **不用于训练**（除非产品政策另有规定，需法务确认）。

## 9. 非功能需求（MVP）


| 类别  | 要求                               |
| --- | -------------------------------- |
| 性能  | 列表页首屏 < 2s（弱网可降级）                |
| 可用性 | 核心流程 3 步内可理解；关键页有空状态引导           |
| 安全  | HTTPS；密码加密存储；基本权限校验              |
| 审计  | 创建人、更新时间（列表可见）                   |
| 兼容  | Web 端优先（Chrome/Edge/Safari 主流版本） |

## 10. 指标与验收（MVP）

### 10.1 北极星（建议）

- **周活跃教师数（WAU）** 或 **人均每周完成「可入库产出」次数**（教案/纪要/评课报告任一）。

### 10.2 过程指标

- 教研：人均纪要生成次数、纪要入库率  
- 教案：草稿→定稿转化率、平均改写次数  
- 评课：报告生成率、提交率

### 10.3 MVP 上线标准（Go-live）

- 三类应用端到端可用 + 资源库可检索 + 工作台可用。  
- 无阻断级缺陷（登录失败、数据丢失、无法保存）。

## 12. 风险与待决事项

1. **校本资源与版权**：上传资料版权责任需用户协议约定。
2. **评课量表**：不同学校差异大，MVP 固定一套需试点确认可接受度。
3. **权限模型**：若未来多校 SaaS，需提前预留租户模型（MVP 可单校）。
4. **AI 幻觉**：必须在教案/评课场景强调审核与引用课标的方式（后续可增强「课标片段检索」）。
